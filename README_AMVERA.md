# Деплой в Amvera Cloud

Нужны **два сервиса** из одного репозитория (папка **`rosstandart_sibur`** в корне Git):

| Сервис | Файл конфигурации | Зачем |
|--------|-------------------|--------|
| **Web Application** | `amvera.yaml` | Браузерный UI, логин, кнопка «Обновить данные сейчас» |
| **Cron Jobs** | `amvera.cron.yaml` | Ежедневный фоновый запуск `main.py` без участия пользователя |

Оба процесса выполняют один и тот же код обновления (`main.py`): cron — по расписанию, web — по кнопке (через тот же интерпретатор, что и Flask).

---

## 1. Общие принципы данных

- Все состояние (кэши, реестр, Excel, **`dashboard.html`**) лежит в каталоге **`DATA_DIR`**.
- Локально по умолчанию это папка **`data/`** рядом с проектом.
- В Amvera в конфигах уже указано **`persistenceMount: /data`** — файлы должны сохраняться на диске платформы.

### Как приложение находит каталог данных

1. Если задана переменная **`DATA_DIR`** — используется она (рекомендуется явно указать **`DATA_DIR=/data`** для Web и для Cron).
2. Иначе, если в окружении **`AMVERA=1`** (типично для Amvera) и существует каталог **`/data`**, используется **`/data`**.
3. Иначе — локальная папка **`data/`** в проекте.

**Важно.** Web и Cron должны видеть **одни и те же файлы**. Это возможно только если:

- к обоим приложениям подключён **один и тот же том** постоянного хранилища на **`/data`**, **или**
- вы явно задаёте одинаковый **`DATA_DIR`** и он указывает на смонтированный общий ресурс.

Если у Cron и Web **разные** изолированные диски, дашборд в браузере не увидит обновления от ночного cron. В этом случае уточните в [документации Amvera](https://docs.amvera.ru/) или у поддержки, как связать одно хранилище с двумя приложениями.

## 2. Web Application (UI + ручное обновление)

1. В панели Amvera создайте приложение типа **Python** / **Web** из вашего Git-репозитория.
2. Укажите **корень проекта** так, чтобы в корне сборки лежали `amvera.yaml`, `requirements.txt`, `web_app.py`, `main.py`, папка `src/`.
3. В настройках сборки/запуска выберите конфиг **`amvera.yaml`** (или переименуйте его в тот файл, который читает ваш проект в Amvera — обычно `amvera.yaml` в корне репозитория).
4. Подключите **постоянное хранилище** с точкой монтирования **`/data`** (как в `run.persistenceMount`).
5. Задайте переменные окружения (в т.ч. **секреты** для пароля):

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `DATA_DIR` | Рекомендуется | **`/data`** — все файлы данных и `dashboard.html` |
| `APP_USERNAME` | Да | Логин технического пользователя |
| `APP_PASSWORD` | Да | Пароль (лучше через «Секрет») |
| `APP_SECRET_KEY` | Да | Длинная случайная строка для сессий Flask |
| `APP_HOST` | Нет | По умолчанию `0.0.0.0` |
| `PORT` | Нет | По умолчанию `8000`; должен совпадать с `containerPort` в `amvera.yaml` |
| `RUN_NOW_TIMEOUT_SECONDS` | Нет | Таймаут ручного запуска `main.py` в секундах (по умолчанию `600`) |

6. После деплоя откройте URL приложения → **`/login`** → дашборд. Кнопка **«Обновить данные сейчас»** запускает **`main.py`** на сервере и перезаписывает **`dashboard.html`** в **`DATA_DIR`**.

Подробнее про Python и `amvera.yaml`: [документация Amvera — конфигурация](https://docs.amvera.ru/applications/configuration/config-file.html).

## 3. Cron Jobs (ежедневное автообновление)

1. Создайте отдельный проект **Cron Jobs** из **того же** репозитория и того же каталога **`rosstandart_sibur`**.
2. Используйте конфиг **`amvera.cron.yaml`** (запуск: **`python3 main.py`** через `scriptName: main.py`).
3. Подключите **то же хранилище** на **`/data`**, что и у Web (или задайте тот же **`DATA_DIR`**).
4. Задайте переменные:

| Переменная | Обязательно |
|------------|-------------|
| `DATA_DIR=/data` | Рекомендуется (должен совпадать с Web) |

Почта для отчётов **не используется** — переменные Gmail не нужны.
5. Расписание задайте в интерфейсе Cron (время на сервере Amvera — **UTC**). Пример: ежедневно в **07:00 UTC** ≈ **10:00 МСК**. Синтаксис см. [Cron Jobs](https://docs.amvera.ru/cron/cronjobs.html).

После срабатывания cron файл **`dashboard.html`** обновится на диске; пользователь при следующем открытии UI увидит актуальные данные (при необходимости обновите страницу в браузере).

---

## 4. Проверка после запуска

1. **Cron:** выполните одно ручное срабатывание в панели (если есть) или дождитесь расписания; в логах должно быть завершение **`main.py`** без ошибки.
2. **Web:** войдите под **`APP_USERNAME` / APP_PASSWORD** → откройте дашборд.
3. Нажмите **«Обновить данные сейчас»** — должно появиться сообщение об успехе; время «Обновлено» в дашборде изменится.
4. Убедитесь, что на **`/data`** (или в **`DATA_DIR`**) появляются/меняются файлы вроде `dashboard_registry.json`, `dashboard.html`.

---

## 5. Частые проблемы

| Симптом | Что проверить |
|---------|----------------|
| «Файл dashboard.html пока не создан» | Ни разу не отработал **`main.py`** (cron или кнопка); нет прав записи в **`DATA_DIR`**. |
| Cron отработал, в UI старые данные | Разные каталоги данных у Web и Cron; выровняйте том **`/data`** и **`DATA_DIR`**. |
| Таймаут по кнопке | Увеличьте **`RUN_NOW_TIMEOUT_SECONDS`** (сеть к rst.gov.ru / fgis.gost.ru может быть медленной). |
| Переменные не подхватились | После изменения env в Amvera обычно нужен **перезапуск** контейнеров ([переменные и секреты](https://docs.amvera.ru/applications/configuration/variables.html)). |

---

## 6. Локальный запуск (для сравнения)

```bash
cd rosstandart_sibur
pip install -r requirements.txt
python main.py          # обновить данные и сгенерировать data/dashboard.html
python web_app.py       # UI на http://localhost:8000
```

Явно задать каталог данных: **`DATA_DIR=/path/to/data python main.py`**.
